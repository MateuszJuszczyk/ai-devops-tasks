# Zadanie 22: Generowanie README dla projektu

## Prompt

**Stwórz prosty plik README.md dla tego projektu, zawierający sekcje: opis, instalacja, użycie, endpointy API.**

## Opis projektu

```text
Nazwa projektu: Simple Task API
Opis: REST API do zarządzania zadaniami, napisane w Node.js z Express i MongoDB.
Główne funkcje:
- Tworzenie, odczytywanie, aktualizacja i usuwanie zadań
- Filtrowanie zadań według statusu i priorytetów
- Prostą autoryzację użytkowników
```

## Rezultat

Wygenerowany plik README zapisano jako `project-readme.md`.

## Uwagi dotyczące jakości dokumentacji

Dokumentacja jest przejrzysta i zawiera podstawowe informacje potrzebne do zrozumienia projektu. Sekcje opis, instalacja, użycie i endpointy API są logicznie uporządkowane.

Najbardziej przydatne są przykłady komend instalacyjnych, przykładowy plik `.env` oraz tabela endpointów API. Dzięki nim nowa osoba w projekcie może szybko uruchomić aplikację i zrozumieć dostępne funkcje.

Dokumentację można ulepszyć przez dodanie dokładnych przykładów odpowiedzi API, instrukcji uruchomienia MongoDB w Dockerze, opisu testów, wymagań systemowych oraz informacji o obsłudze błędów.
